## Cross-Phase Integration (17/18 Wired)

Integration checker verified 18 cross-phase export connections. 17 are fully wired. 1 has a scoped type gap:

### Integration Gap

**`Partial` contract status type mismatch** (PERF-03 related)
- `api/analyze.ts` writes `status: 'Partial'` on global timeout path
- `src/types/contract.ts` defines `status: 'Analyzing' | 'Reviewed' | 'Draft'` — missing `'Partial'`
- No runtime crash (flows through `mapRow<Contract>` without narrowing)
- Dashboard stats exclude Partial contracts (`status === 'Reviewed'` filter)
- No distinct UI state shown to user for partial analysis
- **Severity:** Low — this path only fires on rare 250s timeout scenarios

### Verified Integration Points

- **51→52:** `analysis_usage` table written by pipeline, read by `useAnalysisUsage` hook and Dashboard — RLS policies, numeric coercion, and `run_id` grouping all verified
- **51→53:** Independent migrations touching different columns (`status` CHECK vs `lifecycle_status` column) — no conflict
- **51→54:** Pipeline changes don't affect date extraction — dates still stored in `contract_dates` array
- **52+54→Dashboard:** Cost StatCards in 3-col stat grid, DeadlineTimeline in separate right-column section — no DOM conflict
- **53→AllContracts:** Lifecycle filter is additive `&&` with existing type/sort filters in `useMemo` — no interference
- **53+52→ContractReview:** CostSummaryBar between header and content, LifecycleSelect in header metadata row — separate DOM sections

### E2E Flows Verified (6/6)

1. Upload PDF → two-stage analysis → usage rows written → cost bar appears on review
2. Dashboard loads → fetches analysis_usage → renders cost StatCards + DeadlineTimeline
3. Open contract → see lifecycle badge → change via dropdown → persists to Supabase
4. AllContracts → apply lifecycle filter → contracts narrow → clear restores all
5. Dashboard → click timeline entry → navigates to contract review page
6. Sidebar → badge shows 7-day deadline count → disappears when zero
