  <action>
Create `src/lib/mappers.ts` with two named exports:

1. **`snakeToCamel(s: string): string`** (not exported, internal helper):
   - Converts a single snake_case string to camelCase
   - Implementation: `s.replace(/_([a-z])/g, (_, c) => c.toUpperCase())`
   - Examples: `"risk_score"` -> `"riskScore"`, `"upload_date"` -> `"uploadDate"`, `"id"` -> `"id"` (no change)

2. **`mapRow<T>(row: Record<string, unknown>): T`** (exported):
   - Iterates `Object.entries(row)`
   - For each key: converts key via `snakeToCamel()`
   - For each value: if `value === null`, set to `undefined`; otherwise pass through as-is
   - This means JSONB objects (like `score_breakdown`, `bid_signal`, `pass_results`, `cross_references`, `legal_meta`, `scope_meta`) pass through unchanged because they are already camelCase JSON stored in Postgres
   - Returns `result as T` (type assertion)
   - IMPORTANT: Only convert top-level keys. Do NOT recurse into nested objects/arrays.

3. **`mapRows<T>(rows: Record<string, unknown>[]): T[]`** (exported):
   - Implementation: `rows.map((row) => mapRow<T>(row))`

File must use named exports only (no default export). Follow existing pattern from `src/lib/supabase.ts`.
