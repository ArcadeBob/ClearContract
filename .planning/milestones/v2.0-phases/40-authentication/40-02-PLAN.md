**2. Modify `src/App.tsx`** -- Add auth gate with state clearing on sign-out:

Add imports:
```typescript
import { useAuth } from './contexts/AuthContext';
import { LoginPage } from './pages/LoginPage';
import { LoadingScreen } from './components/LoadingScreen';
```

Extract the authenticated app content into a separate inner component called `AuthenticatedApp`. Move ALL existing App logic (hooks like `useContractStore`, `useRouter`, `useCompanyProfile`, state, handlers, and the JSX return) into `AuthenticatedApp`. This component is what renders when the user is authenticated.

The outer `App` function becomes a thin auth gate:
```typescript
export function App() {
  const { session, isLoading, signOut } = useAuth();

  if (isLoading) {
    return <LoadingScreen />;
  }

  if (!session) {
    return <LoginPage />;
  }

  return <AuthenticatedApp signOut={signOut} />;
}
```

Define `AuthenticatedApp` as a separate function component (in the same file, above `App`):
```typescript
function AuthenticatedApp({ signOut }: { signOut: () => Promise<void> }) {
  // All existing App body goes here: useContractStore, useRouter, handlers, JSX
  // ...
}
```

**Why this approach clears in-memory state on sign-out (per CONTEXT.md locked decision):** When session becomes null after sign-out, `App` returns `<LoginPage />` instead of `<AuthenticatedApp />`. React unmounts `AuthenticatedApp`, which destroys all its `useState` hooks including `useContractStore`'s contracts array and `useRouter`'s navigation state. When the user signs in again, `AuthenticatedApp` remounts fresh with clean state. No explicit reset logic needed.

Pass `signOut` to Sidebar within `AuthenticatedApp`. Update the Sidebar JSX to add the `onSignOut` prop:
```tsx
<Sidebar
  activeView={activeView}
  onNavigate={(view) => navigateTo(view)}
  contractCount={contracts.length}
  onSignOut={signOut}
/>
```
