**1. Create `src/lib/supabase.ts`** -- Supabase client singleton:
```typescript
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL;
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY;

if (!supabaseUrl || !supabaseAnonKey) {
  throw new Error('Missing VITE_SUPABASE_URL or VITE_SUPABASE_ANON_KEY environment variables');
}

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

**2. Create `src/contexts/AuthContext.tsx`** -- Auth provider and hook:
- Export `AuthProvider` component and `useAuth` hook (named exports, no defaults)
- State: `session: Session | null` (init `null`), `isLoading: boolean` (init `true`)
- In `useEffect`, call `supabase.auth.onAuthStateChange((event, session) => { ... })`:
  - Set session to `null` on `SIGNED_OUT` event, otherwise set to the session value
  - Set `isLoading` to `false` on every event (this covers the `INITIAL_SESSION` event for AUTH-06)
  - Return cleanup: `subscription.unsubscribe()`
- `signOut` async function: calls `await supabase.auth.signOut()`
- Context value type: `{ session: Session | null; isLoading: boolean; signOut: () => Promise<void> }`
- `useAuth()` hook throws `Error('useAuth must be used within AuthProvider')` if context is undefined
- Import `Session` type from `@supabase/supabase-js`
- Do NOT use async callbacks inside `onAuthStateChange` (anti-pattern per Supabase docs)

     - `label` with `htmlFor="password"`, class `block text-sm text-slate-700 mb-1`, text "Password"
     - `input` with `id="password"`, `type="password"`, `required`, no placeholder
     - Same input classes as email
   - Error message (conditional, shown only when `error` state is truthy):
     - `p` with `role="alert"` class `text-sm text-red-600 mt-2`, displays the error string
   - Submit button (`mt-8` for 32px gap from last field):
     - `button` with `type="submit"`, disabled when `isSubmitting` or email/password empty
     - Classes: `w-full py-2.5 bg-slate-900 text-white text-sm font-semibold rounded-lg hover:bg-slate-800 transition-colors disabled:opacity-50 disabled:cursor-not-allowed`

State: `email` (string), `password` (string), `error` (string | null), `isSubmitting` (boolean).

`handleSubmit` function:
- `e.preventDefault()`
- Set `isSubmitting = true`, `error = null`
- Call `const { error } = await supabase.auth.signInWithPassword({ email, password })`
- If error: set error state to `'Invalid email or password'` (generic message per CONTEXT.md security decision)
- Catch block for network errors: set error state to `'Unable to connect. Please try again.'`
- Finally: `setIsSubmitting(false)`
- Do NOT handle success -- `onAuthStateChange` in AuthContext handles session update

Clear error when user modifies either field: add `onChange` handlers that call `setError(null)` alongside updating the field value.

Accessibility: `autoFocus` on email input for focus management on mount.
