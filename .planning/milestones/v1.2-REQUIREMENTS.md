# Requirements: ClearContract v1.2 UX Foundations

**Defined:** 2026-03-12
**Core Value:** Upload a contract, walk away with a complete breakdown -- risks, scope, dates, compliance -- with exact contract language quoted so you can act immediately.

## v1.2 Requirements

Requirements for UX Foundations milestone. Each maps to roadmap phases.

### Data Persistence

- [ ] **PERSIST-01**: Analyzed contracts persist in localStorage across browser sessions (page refresh does not lose data)
- [ ] **PERSIST-02**: Contract data serialization handles all fields including findings, dates, bidSignal, and passResults
- [ ] **PERSIST-03**: Mock contracts load only on first visit; subsequent visits use persisted data
- [ ] **PERSIST-04**: Storage quota handling -- graceful degradation when localStorage is full (warn user, don't crash)

### Contract Management

- [ ] **MGMT-01**: User can delete a contract from the contracts list with confirmation prompt
- [ ] **MGMT-02**: Deleting a contract removes it from both state and localStorage
- [ ] **MGMT-03**: If the active/reviewed contract is deleted, navigate to dashboard

### Upload Error Feedback

- [ ] **UX-01**: File type rejection (non-PDF) shows inline error message in UploadZone
- [ ] **UX-02**: File size rejection (over limit) shows inline error message with size details
- [ ] **UX-03**: API analysis failure shows a toast/banner notification (not buried as a finding)
- [ ] **UX-04**: Network timeout or connection failure shows user-friendly error with retry option

### Empty States & Dashboard Polish

- [ ] **DASH-01**: Dashboard shows welcoming empty state with upload CTA when 0 contracts exist
- [ ] **DASH-02**: DateTimeline shows "No dates extracted" message when dates array is empty
- [ ] **DASH-03**: Dashboard stat trends are hidden (not hardcoded fakes) until real historical data exists
- [ ] **DASH-04**: ContractCard displays the numeric risk score (0-100) alongside finding counts

## Out of Scope

| Feature | Reason |
|---------|--------|
| Supabase/backend persistence | localStorage sufficient for single-user; backend is v2 |
| Multi-user / auth | Single-user app for now |
| Undo/redo | Over-engineering for current needs |
| Bulk operations | No need until 50+ contracts |
| IndexedDB | localStorage handles contract JSON fine under 5MB |
| Export/Share buttons | Sprint 2 feature |
