**ContractReview.tsx — Hide-resolved toggle:**

1. Add localStorage-backed state for hide-resolved:
   ```typescript
   const HIDE_RESOLVED_KEY = 'clearcontract:hide-resolved';
   const [hideResolved, setHideResolved] = useState(() => {
     try {
       return localStorage.getItem(HIDE_RESOLVED_KEY) === 'true';
     } catch {
       return false;
     }
   });

   const toggleHideResolved = () => {
     setHideResolved((prev) => {
       const next = !prev;
       try { localStorage.setItem(HIDE_RESOLVED_KEY, String(next)); } catch {}
       return next;
     });
   };
   ```

2. **Apply hide-resolved filter at the data level** (CRITICAL — per Pitfall 3 from research, filter BEFORE both rendering paths diverge):
   ```typescript
   const visibleFindings = hideResolved
     ? contract.findings.filter((f) => !f.resolved)
     : contract.findings;
   ```
   Then replace ALL references to `contract.findings` in `groupedFindings` and `flatFindings` computations with `visibleFindings`. Keep `contract.findings` for counts (total/resolved) since those should reflect ALL findings regardless of filter.
