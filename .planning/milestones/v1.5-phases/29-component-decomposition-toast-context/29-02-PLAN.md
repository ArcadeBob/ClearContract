       - Import `Toast` from '../components/Toast' (keep Toast as separate presentational component)

       - Export interface `ToastContextValue`:
         ```
         showToast: (opts: { type: 'error' | 'warning' | 'info' | 'success'; message: string; onRetry?: () => void }) => void;
         ```

       - Export `ToastContext = createContext<ToastContextValue | null>(null)`

       - Export `ToastProvider` component ({ children: ReactNode }):
         a. State: `toast` of type `{ type, message, onRetry } | null`, initialized to null
         b. Use a `useRef` for the auto-dismiss timer to clean up properly
         c. `showToast` callback (memoized with useCallback):
            - Sets toast state to the new toast (replace-on-new semantics)
            - Clears any existing timer via the ref
            - If no `onRetry`, starts a 3-second auto-dismiss timer (per user decision -- changed from existing 8 seconds)
            - If `onRetry` is present, do NOT auto-dismiss (preserve existing behavior)
         d. `dismissToast` callback: sets toast to null, clears timer
         e. useEffect cleanup: clear timer on unmount
         f. Renders:
            ```
            <ToastContext.Provider value={{ showToast }}>
              {children}
              <AnimatePresence>
                {toast && (
                  <Toast
                    type={toast.type}
                    message={toast.message}
                    onRetry={toast.onRetry}
                    onDismiss={dismissToast}
                  />
                )}
              </AnimatePresence>
            </ToastContext.Provider>
            ```

       IMPORTANT: The Toast component currently uses `absolute` positioning relative to its parent. Since ToastProvider will render outside the `<main>` element, change the Toast's position to `fixed` when rendered by the provider. Two approaches (pick the simpler one):
       - Option A: Modify Toast.tsx to use `fixed` instead of `absolute` (simpler, since after migration ALL toasts come from the provider)
       - Option B: Leave Toast.tsx unchanged and wrap it in a positioned container in ToastProvider

       Use Option A: update Toast.tsx to change `absolute` to `fixed` on line 51. This is safe because after plan 03 wires everything, ALL toast rendering goes through ToastProvider. Document this change.
