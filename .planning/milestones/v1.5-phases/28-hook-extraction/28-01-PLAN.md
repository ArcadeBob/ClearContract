  <action>
Create `src/hooks/useInlineEdit.ts` per the locked API in CONTEXT.md:
- Options interface: `{ initialValue: string; autoFocus?: boolean; validate?: (value: string) => string; onSave: (value: string) => void }`
- Return interface: `{ isEditing, editValue, setEditValue, startEditing, commitEdit, cancelEdit, onKeyDown, inputRef }`
- `startEditing` copies initialValue into editValue and sets isEditing true
- `commitEdit` applies validate transform if provided, calls onSave if value changed, sets isEditing false. If validate returns empty string, skip onSave (empty = invalid)
- `cancelEdit` reverts editValue to initialValue AND sets isEditing false
- `onKeyDown` handles Enter (commitEdit) and Escape (cancelEdit)
- useEffect auto-focuses and selects inputRef when isEditing becomes true AND autoFocus is true
- Use useCallback for startEditing, commitEdit, cancelEdit, onKeyDown with correct exhaustive-deps
- inputRef typed as RefObject<HTMLInputElement> -- FindingCard will simply not use it (textarea with buttons)

Create `src/hooks/useContractFiltering.ts`:
- Move `CATEGORY_ORDER` and `severityRank` constants from ContractReview.tsx into this file as module-level constants
- Move `ACTION_PRIORITIES` constant as well
- Options interface: `{ findings: Finding[] }`
- FilterState type: `{ severities: Set<Severity>; categories: Set<Category>; priorities: Set<string>; negotiationOnly: boolean }`
- Return interface: `{ filters, toggleFilter, resetFilters, hideResolved, toggleHideResolved, visibleFindings, groupedFindings, flatFindings }`
- Initialize filter Sets to full (all severities, all categories, all priorities selected)
- `toggleFilter(type, value?)` -- for 'severities'|'categories'|'priorities': toggle value in/out of the Set. For 'negotiationOnly': toggle the boolean (value param ignored)
- `resetFilters()` -- reset all Sets to full, negotiationOnly to false
- `hideResolved` initialized from `loadRaw('clearcontract:hide-resolved').data === 'true'`
- `toggleHideResolved` toggles and persists via `saveRaw('clearcontract:hide-resolved', String(next))`
- `visibleFindings` computed via useMemo -- same filter logic currently in ContractReview lines 170-188
- `groupedFindings` computed via useMemo -- same logic as ContractReview lines 191-203 (CATEGORY_ORDER map, filter empty, sort by max severity then count)
- `flatFindings` computed via useMemo -- severity-sorted copy of visibleFindings
- All useMemo/useCallback must have correct exhaustive-deps

Both hooks: export as named exports (not default), follow existing one-hook-per-file convention.
