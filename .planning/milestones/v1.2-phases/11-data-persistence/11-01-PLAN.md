  <name>Task 1: Create contract storage utility</name>
  <files>src/storage/contractStorage.ts</files>
  <action>
Create `src/storage/contractStorage.ts` following the profileLoader.ts pattern.

Constants:
- `CONTRACTS_STORAGE_KEY = 'clearcontract:contracts'`
- `SEEDED_KEY = 'clearcontract:contracts-seeded'`

Functions:

1. `loadContracts(): { contracts: Contract[]; fromStorage: boolean }`
   - Try `localStorage.getItem(CONTRACTS_STORAGE_KEY)`
   - If found and parses as array: return `{ contracts: parsed, fromStorage: true }`
   - If not found: check `localStorage.getItem(SEEDED_KEY)`
     - If SEEDED_KEY not set: this is first visit. Import MOCK_CONTRACTS, save them via saveContracts, set SEEDED_KEY to `'true'`, return `{ contracts: MOCK_CONTRACTS, fromStorage: false }`
     - If SEEDED_KEY is set: user has been here before but cleared data. Return `{ contracts: [], fromStorage: true }`
   - On any error (parse fail, localStorage unavailable): return `{ contracts: [], fromStorage: false }` -- do NOT throw

2. `saveContracts(contracts: Contract[]): { success: boolean; error?: string }`
   - Try `localStorage.setItem(CONTRACTS_STORAGE_KEY, JSON.stringify(contracts))`
   - On success: return `{ success: true }`
   - On DOMException (QuotaExceededError): return `{ success: false, error: 'Storage is full. Your contracts are safe in memory but will not persist after refresh.' }`
   - On other error: return `{ success: false, error: 'Could not save to browser storage.' }`

No Date object serialization needed -- all Contract fields are already JSON-safe (strings, numbers, plain objects). The discriminated unions (LegalMeta, ScopeMeta) serialize as plain objects automatically.

Do NOT import from React. This is a standalone utility module (same pattern as profileLoader.ts).

<task type="auto">
  <name>Task 2: Refactor useContractStore with persistence and quota warning</name>

  <action>
**useContractStore.ts refactor:**

1. Replace `import { MOCK_CONTRACTS }` with `import { loadContracts, saveContracts } from '../storage/contractStorage'`

2. Add a `storageWarning` state: `const [storageWarning, setStorageWarning] = useState<string | null>(null)`

3. Initialize contracts from storage:
   ```
   const [contracts, setContracts] = useState<Contract[]>(() => {
     const { contracts } = loadContracts();
     return contracts;
   });
   ```

4. Create a helper `persistAndSet` that wraps every contracts mutation:
   ```
   const persistAndSet = (updater: (prev: Contract[]) => Contract[]) => {
     setContracts(prev => {
       const next = updater(prev);
       const result = saveContracts(next);
       if (!result.success) {
         setStorageWarning(result.error ?? 'Storage error');
       } else {
         setStorageWarning(null);
       }
       return next;
     });
   };
   ```

5. Rewrite `addContract` to use `persistAndSet`:
   ```
   const addContract = (contract: Contract) => {
     persistAndSet(prev => [contract, ...prev]);
   };
   ```

6. Rewrite `updateContract` to use `persistAndSet`:
   ```
   const updateContract = (id: string, updates: Partial<Contract>) => {
     persistAndSet(prev => prev.map(c => c.id === id ? { ...c, ...updates } : c));
   };
   ```

7. Add `deleteContract` (for Phase 12 prep):
   ```
   const deleteContract = (id: string) => {
     persistAndSet(prev => prev.filter(c => c.id !== id));
   };
   ```

8. Add `dismissStorageWarning`:
   ```
   const dismissStorageWarning = () => setStorageWarning(null);
   ```

9. Return object adds: `deleteContract`, `storageWarning`, `dismissStorageWarning`

**App.tsx changes:**

1. Destructure `storageWarning` and `dismissStorageWarning` from `useContractStore()`

2. Add a warning banner inside the return JSX, just before `<main>` (after the `<Sidebar>` component):
   ```tsx
   {storageWarning && (
     <div className="fixed top-4 right-4 z-50 max-w-md bg-amber-50 border border-amber-200 rounded-lg p-4 shadow-lg flex items-start gap-3">
       <span className="text-amber-600 text-sm flex-1">{storageWarning}</span>
       <button
         onClick={dismissStorageWarning}
         className="text-amber-400 hover:text-amber-600 text-lg leading-none"
         aria-label="Dismiss warning"
       >
         &times;
       </button>
     </div>
   )}
   ```

Do NOT change any other App.tsx logic (upload handler, view routing, etc.).
