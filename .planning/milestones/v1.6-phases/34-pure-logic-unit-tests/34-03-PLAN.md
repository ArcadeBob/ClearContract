5. Assert `sourcePass` equals the pass name

Example for indemnification:
```typescript
it('converts legal-indemnification findings with legalMeta', () => {
  const finding = createIndemnificationFinding({ riskType: 'broad', hasInsuranceGap: true });
  const result = mergePassResults(
    [fulfilled('legal-indemnification', [finding])],
    [pass('legal-indemnification')]
  );
  expect(result.findings).toHaveLength(1);
  const f = result.findings[0];
  expect(f.legalMeta?.clauseType).toBe('indemnification');
  expect(f.legalMeta && 'riskType' in f.legalMeta && f.legalMeta.riskType).toBe('broad');
  expect(f.legalMeta && 'hasInsuranceGap' in f.legalMeta && f.legalMeta.hasInsuranceGap).toBe(true);
  expect(f.sourcePass).toBe('legal-indemnification');
});
```

**describe('mergePassResults - deduplication'):**

clauseRef+category composite key (3 tests):
1. Specialized vs generic same clauseRef+category: specialized wins.
   - Create a finding from `createIndemnificationFinding({ clauseReference: 'Section 5.1' })` via legal-indemnification pass.
   - Create a generic finding `{ severity: 'Critical', category: 'Legal Issues', title: 'Same Issue', clauseReference: 'Section 5.1', description: 'x', recommendation: 'x' }` via risk-overview pass.
   - Pass both. Output should have 1 finding with legalMeta (the specialized one).

2. Same pass type, different severity, same clauseRef+category: higher severity wins.
   - Two indemnification findings with same clauseRef, one Critical, one Medium. Output keeps Critical.

   - Two findings with different clauseReferences. Output has 2 findings.

Title-based fallback (2 tests):
