```typescript
import { z } from 'zod';

const SeverityEnum = z.enum(['Critical', 'High', 'Medium', 'Low', 'Info']);
const CategoryEnum = z.enum([
  'Legal Issues', 'Scope of Work', 'Contract Compliance', 'Labor Compliance',
  'Insurance Requirements', 'Important Dates', 'Financial Terms',
  'Technical Standards', 'Risk Assessment',
]);
const DateTypeEnum = z.enum(['Start', 'Milestone', 'Deadline', 'Expiry']);

export const FindingSchema = z.object({
  severity: SeverityEnum,
  category: CategoryEnum,
  title: z.string(),
  description: z.string(),
  recommendation: z.string(),
  clauseReference: z.string(),
  clauseText: z.string().optional(),
  explanation: z.string().optional(),
});

export const ContractDateSchema = z.object({
  label: z.string(),
  date: z.string(),
  type: DateTypeEnum,
});

export const PassResultSchema = z.object({
  findings: z.array(FindingSchema),
  dates: z.array(ContractDateSchema),
});

export const RiskOverviewResultSchema = z.object({
  client: z.string(),
  contractType: ContractTypeEnum,
  findings: z.array(FindingSchema),
  dates: z.array(ContractDateSchema),
});
```

**Add 4 new passes to the ANALYSIS_PASSES array** (append after the existing 3):

1. **legal-indemnification** pass:
   - `isOverview: false, isLegal: true, schema: IndemnificationPassResultSchema`
   - System prompt must include: analyzing from glazing/glass installation subcontractor perspective, finding ALL indemnification and hold-harmless clauses, quoting EXACT COMPLETE clause text verbatim (no truncation, no paraphrase), classifying by type (broad/intermediate/limited) with detection signals for each type, checking for insurance gaps against standard glazing sub CGL/umbrella coverage, explaining in plain English why the clause matters to a glazing sub compared to the standard/fair version, noting cross-references, including state-specific enforceability context if governing law state is specified in the contract
   - MANDATORY severity rules baked into prompt: Broad form = Critical, Intermediate = High, Limited = Medium
   - Missing-clause detection: flag if contract only has Sub->GC indemnification with no mutual indemnification
   - One finding per clause instance
   - User prompt: "Analyze all indemnification and hold-harmless clauses in this glazing subcontract."

2. **legal-payment-contingency** pass:
   - `isOverview: false, isLegal: true, schema: PaymentContingencyPassResultSchema`
   - System prompt must include: finding ALL pay-if-paid and pay-when-paid provisions, detection signals ("contingent upon", "condition precedent to payment", "only if and to the extent" for pay-if-paid; "within X days of receipt", "upon receipt of payment from owner" for pay-when-paid), quoting exact text, explaining enforceability by state if governing law specified (note ~13 states prohibit pay-if-paid), plain-English explanation of risk from glazing sub perspective, cross-references
   - MANDATORY severity: pay-if-paid = Critical, pay-when-paid = High
   - Missing-clause detection: if no payment timeline specified, flag it
   - User prompt: "Analyze all payment contingency clauses (pay-if-paid, pay-when-paid) in this glazing subcontract."

3. **legal-liquidated-damages** pass:
   - `isOverview: false, isLegal: true, schema: LiquidatedDamagesPassResultSchema`
   - System prompt must include: finding ALL liquidated damages clauses, extracting amount/rate, assessing proportionality against contract value, determining if capped or uncapped, checking for flow-through from owner LD provisions, quoting exact text, plain-English explanation of financial exposure for glazing sub, cross-references
   - MANDATORY severity: any LD clause = High minimum, uncapped or disproportionate to contract value = Critical
   - User prompt: "Analyze all liquidated damages clauses in this glazing subcontract."

4. **legal-retainage** pass:
   - `isOverview: false, isLegal: true, schema: RetainagePassResultSchema`
   - System prompt must include: finding ALL retainage/retention provisions, extracting percentage, release conditions, determining if release tied to sub's work completion or overall project completion, checking for reduction after 50% completion, quoting exact text, plain-English explanation of cash flow impact for glazing sub, cross-references
   - MANDATORY severity: tied to overall project completion (not sub's work) = High, above 10% = High, standard 5-10% tied to sub's substantial completion = Low, missing release conditions = Critical
   - User prompt: "Analyze all retainage and retention provisions in this glazing subcontract."

ALL prompts must include these common instructions:
- "Quote the EXACT, COMPLETE clause text as it appears in the contract. Do not truncate, paraphrase, summarize, or reconstruct. Include the full paragraph/section."
- "Always include the section/article number in clauseReference"
- "If the contract specifies a governing law state, include state-specific enforceability context. If no state is specified, note that enforceability varies by jurisdiction."
- "Contrast each clause with what a standard/fair version looks like so the user can see what is abnormal"
- "Explain why this matters specifically to a glazing/glass installation subcontractor, not generic legal language"
- "Do NOT assess or provide a risk score -- that is computed separately"
- For missing-clause findings: "Use clauseReference 'Not Found' and clauseText 'N/A - Protective clause absent' when flagging a missing protective clause. Search ALL sections of the contract before concluding a clause is absent."

**Update `runAnalysisPass`** to use the pass-specific schema when available:

```typescript
const outputFormat = pass.schema
```

**Add a `convertLegalFinding` function** that maps type-specific fields from legal pass results into the unified Finding shape with `legalMeta`. This function takes the raw parsed finding object and the pass name, returns a Finding-shaped object with legalMeta populated. Follow the pattern from 02-RESEARCH.md Code Examples section.

**Enhance `mergePassResults`**:

1. Tag each finding with `sourcePass` during merge (set to `pass.name`).

2. Replace title-only deduplication with composite key dedup:

   - Among same-type passes, keep the one with higher severity
   - Fallback: title-based dedup remains as secondary pass for findings without clauseReference or for non-legal findings

3. For legal pass results, use `convertLegalFinding` to map metadata into legalMeta before adding to allFindings.

4. Keep the existing client/contractType extraction from risk-overview pass unchanged.

5. Keep `computeRiskScore` unchanged -- the severity-based scoring already works with the new legal findings.

**Do NOT modify existing pass prompts** (risk-overview, dates-deadlines, scope-financial). The dedup strategy handles overlap.
