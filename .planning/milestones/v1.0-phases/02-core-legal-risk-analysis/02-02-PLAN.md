**Create `src/components/LegalMetaBadge.tsx`:**

A component that renders clause-type-specific metadata as compact, readable badges/pills. Props: `meta: LegalMeta` (import LegalMeta from `../types/contract`).

Render different content based on `meta.clauseType` discriminant:

Badge/pill base classes: `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium mr-2`.

Wrap the whole thing in a `div` with `flex flex-wrap items-center gap-1 mt-2`.

Export as named export: `export function LegalMetaBadge(...)`.

  <name>Task 2: Update FindingCard to display clause text, explanation, and legal metadata</name>

  <action>
Update `FindingCard.tsx` to conditionally render the new legal finding details. The existing layout (category icon, severity badge, title, description, recommendation, clause reference) stays exactly as-is. New sections are added BETWEEN the recommendation block and the clause reference line.

**Import new components:**
```typescript
import { ClauseQuote } from './ClauseQuote';
import { LegalMetaBadge } from './LegalMetaBadge';
```

**Add these conditional sections after the recommendation block and before the clauseReference footer:**

1. **Clause text** (when `finding.clauseText` exists AND is not 'N/A - Protective clause absent'):
   ```tsx
   {finding.clauseText && finding.clauseText !== 'N/A - Protective clause absent' && (
     <ClauseQuote text={finding.clauseText} reference={finding.clauseReference || ''} />
   )}
   ```

2. **Plain-English explanation** (when `finding.explanation` exists):
   ```tsx
   {finding.explanation && (
     <div className="bg-amber-50 border border-amber-100 rounded-md p-3 mb-3">
       <p className="text-xs font-semibold text-amber-700 uppercase tracking-wide mb-1">
         Why This Matters
       </p>
       <p className="text-sm text-amber-800 leading-relaxed">{finding.explanation}</p>
     </div>
   )}
   ```
   Style matches the existing recommendation block pattern (bg-blue-50) but uses amber to distinguish explanation from recommendation.

3. **Legal metadata badges** (when `finding.legalMeta` exists):
   ```tsx
   {finding.legalMeta && <LegalMetaBadge meta={finding.legalMeta} />}
   ```

4. **Cross-references** (when `finding.crossReferences` exists and has items):
   ```tsx
   {finding.crossReferences && finding.crossReferences.length > 0 && (
     <div className="flex flex-wrap items-center gap-1 mt-2 mb-2">
       <span className="text-xs font-medium text-slate-500">See also:</span>
       {finding.crossReferences.map((ref, i) => (
         <span key={i} className="inline-flex items-center px-2 py-0.5 rounded-full text-xs bg-slate-100 text-slate-600">
           {ref}
         </span>
       ))}
     </div>
   )}
   ```

**Layout order within FindingCard (top to bottom):**
1. Category icon + severity badge (existing, unchanged)
2. Title (existing, unchanged)
3. Description (existing, unchanged)
4. Recommendation block (existing, unchanged)
5. Clause text quote block (NEW -- conditional)
6. Explanation block (NEW -- conditional)
7. Legal metadata badges (NEW -- conditional)
8. Cross-references (NEW -- conditional)

All new sections are conditional -- they only render when the relevant field exists. Non-legal findings (which lack clauseText, explanation, legalMeta, crossReferences) render exactly as before with zero visual change.

Do NOT add any new framer-motion animations to the new sections. They inherit the parent card's entrance animation.
