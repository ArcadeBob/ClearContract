1. In `src/types/contract.ts`, add two optional fields to the `Finding` interface:
   - `resolved?: boolean` — toggle state for finding resolution
   - `note?: string` — user-authored annotation text

   Add them after `downgradedFrom` (last current field). Do NOT add defaults or migration logic — nullish coalescing handles backward compatibility with existing localStorage data.

2. In `src/hooks/useContractStore.ts`, add two new methods using the existing `persistAndSet` pattern:

   ```typescript
   const toggleFindingResolved = (contractId: string, findingId: string) => {
     persistAndSet((prev) =>
       prev.map((c) =>
         c.id === contractId
           ? {
               ...c,
               findings: c.findings.map((f) =>
                 f.id === findingId ? { ...f, resolved: !f.resolved } : f
               ),
             }
           : c
       )
     );
   };

   const updateFindingNote = (contractId: string, findingId: string, note: string | undefined) => {
     persistAndSet((prev) =>
       prev.map((c) =>
         c.id === contractId
           ? {
               ...c,
               findings: c.findings.map((f) =>
                 f.id === findingId ? { ...f, note } : f
               ),
             }
           : c
       )
     );
   };
   ```

   Add both methods to the return object of `useContractStore`.
  </action>
