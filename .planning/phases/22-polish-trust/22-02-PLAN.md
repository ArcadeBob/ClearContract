**UX-05 -- Upcoming Deadlines widget replacing compliance card:**

In `Dashboard.tsx`:
1. Find the static "Compliance Update" card (or similar static card).
2. Replace it with an "Upcoming Deadlines" widget.
3. Compute upcoming dates using useMemo with contracts as dependency:
   ```typescript
   const upcomingDates = useMemo(() => {
     const now = new Date();
     now.setHours(0, 0, 0, 0);
     return contracts
       .filter(c => c.status === 'Reviewed')
       .flatMap(c => c.dates.map(d => ({ ...d, contractId: c.id, contractName: c.name })))
       .filter(d => {
         const target = new Date(d.date);
         target.setHours(0, 0, 0, 0);
         return target.getTime() >= now.getTime() - 86400000; // Include today and future
       })
       .sort((a, b) => new Date(a.date).getTime() - new Date(b.date).getTime())
       .slice(0, 5);
   }, [contracts]);
   ```
4. Render each date with urgency coloring (same rules as UX-03: red <=7 days, amber <=30 days, green >30 days, muted for past). Include contract name, date label, date value, and days-away text.
5. Each deadline item should be clickable, navigating to that contract's review page via `navigateTo('review', contractId)`.
6. If no upcoming dates, show an empty state: "No upcoming deadlines" with a calendar icon.

Create a helper function `getDateUrgency(dateStr: string)` that returns `{ label: string; colorClass: string; isPast: boolean }` for reuse. Place it in Dashboard.tsx for now (Plan 03 will also need it for DateTimeline -- if duplication occurs, Plan 03 can extract to a shared util).
