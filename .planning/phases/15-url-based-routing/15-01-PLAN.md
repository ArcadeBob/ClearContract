  <action>
Create `src/hooks/useRouter.ts` implementing a custom History API router hook (~30-50 lines):

1. `parseUrl(pathname: string): RouterState` function:
   - `/contracts/:id` -> `{ view: 'review', contractId: id }` (use `startsWith('/contracts/')` + slice, not regex)
   - `/settings` -> `{ view: 'settings', contractId: null }`
   - Everything else -> `{ view: 'dashboard', contractId: null }`

2. `useRouter()` hook:
   - Initialize state with `useState(() => parseUrl(window.location.pathname))` (reads URL on mount for refresh/deep link support -- ROUTE-02, ROUTE-03)
   - `useEffect` with `popstate` event listener that calls `setState(parseUrl(window.location.pathname))` (back/forward support -- ROUTE-01). Do NOT call `pushState` inside the popstate handler.
   - `navigateTo(view: ViewState, contractId?: string)` callback:
     - For `view === 'upload'`: set state to `{ view: 'upload', contractId: null }` WITHOUT calling `pushState` (upload is transient, no URL per user decision)
     - For `view === 'review' && contractId`: push `/contracts/${contractId}`
     - For `view === 'settings'`: push `/settings`
     - For everything else (`dashboard`, `contracts`): push `/`
     - After `pushState`, call `setState` with the new state (pushState does NOT trigger popstate)
   - Return `{ activeView: state.view, activeContractId: state.contractId, navigateTo }`

Update `vercel.json` to add SPA catch-all rewrites. The `/api` rewrite MUST come first to prevent the catch-all from intercepting API calls (Pitfall 1 from research):

```json
  },
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/$1" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
