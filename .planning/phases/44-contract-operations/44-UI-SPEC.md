## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | custom Tailwind components (no headless library) |
| Icon library | Lucide React |
| Font | Inter (300, 400, 500, 600, 700) via Google Fonts |

**Note:** This phase adds no new UI components. All user-facing interactions (delete dialog, resolve toggle, note editor, inline rename) already exist. ## Copywriting Contract

### Error Toasts (Rollback Feedback)

| Operation | Toast Message | Toast Type |
|-----------|---------------|------------|
| Delete contract fails | "Failed to delete contract. Changes reverted." | error |
| Resolve/unresolve finding fails | "Failed to update finding. Changes reverted." | error |
| Save note fails | "Failed to save note. Changes reverted." | error |
| Delete note fails | "Failed to delete note. Changes reverted." | error |
| Rename contract fails | "Failed to rename contract. Changes reverted." | error |
| Re-analyze preservation batch partial fail | (silent — console.error only, no user toast) | n/a |

**Pattern:** "Failed to [verb] [noun]. Changes reverted." No retry button. Auto-dismiss after 3 seconds (existing ToastProvider default). User can manually dismiss via X button.

## Interaction Contracts

### Optimistic Update Visual Behavior

All five mutations follow the same visual contract:

1. **User triggers action** — UI updates instantly (no spinner, no loading state)
2. **Success (background)** — no visible feedback; the optimistic state is already correct
3. **Failure (background)** — UI reverts to pre-mutation state; error toast appears at top-center

### Delete Contract

| Step | Visual Behavior |
|------|----------------|
| User clicks delete | ConfirmDialog opens (portal, `z-50`, backdrop `bg-black/50`) |
| Cancel button auto-focused | Cancel is the safe default (existing behavior) |
| User confirms | Dialog closes, contract removed from list immediately |
| Success | No additional feedback; contract is gone from all views |
| Failure | Contract reappears in list; error toast shown |
| Post-delete navigation | If user was on deleted contract's review page, navigate to dashboard |

### Resolve/Unresolve Finding

| Step | Visual Behavior |
|------|----------------|
| User clicks check icon | Finding opacity toggles immediately (`opacity-60` for resolved), title gets `line-through` |
| Icon swaps | Unresolved: gray `Check` icon (`text-slate-300`). Resolved: green `CheckCircle2` icon (`text-emerald-500`) |
| Failure | Finding reverts to previous resolved state; error toast shown |

### Finding Note CRUD

| Step | Visual Behavior |
|------|----------------|
| Add: user clicks "+ Add note" | Textarea appears below finding with Cancel/Save buttons |
| Edit: user clicks pencil icon | Textarea replaces note display with current text pre-filled |
| Save: user clicks Save, presses Enter, or blurs textarea | Textarea closes, note displays in `violet-50` card |
| Delete: user clicks X icon | ConfirmDialog opens; on confirm, note removed from display |
| Failure (save/delete) | Note reverts to previous state; error toast shown |
| Textarea focus ring | `ring-violet-300` and `border-violet-300` on focus |
| Save button disabled state | `opacity-50` when textarea is empty/whitespace-only |
| Character limit | 1000 characters (`maxLength={1000}`) |

### Inline Rename

| Step | Visual Behavior |
|------|----------------|
| User activates edit | Contract name becomes editable input (existing `useInlineEdit` pattern) |
| Save on blur | Input loses focus, name updates immediately |
| Save on Enter | Same as blur save |
| Cancel on Escape | Input reverts to original name, exits edit mode |
| Failure | Name reverts to pre-edit value; error toast shown |

### Re-analyze with Preservation

| Step | Visual Behavior |
|------|----------------|
| User clicks re-analyze | ConfirmDialog opens with info icon (blue `RefreshCw`, not red warning) |
| User confirms | Dialog closes, analysis progress UI appears (existing `AnalysisProgress` component) |
| Server completes | New findings merge with preserved resolved/notes; review page updates |
| Preservation write fails (partial) | Silent — findings display correctly from in-memory merge; console.error only |
| Server fails entirely | Contract reverts to pre-analyze state (snapshot restore); error toast shown |
