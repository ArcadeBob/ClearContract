**Initialization (useState lazy init):**
- Try `localStorage.getItem(STORAGE_KEY)`.
- If found, parse and spread: `{ ...DEFAULT_COMPANY_PROFILE, ...JSON.parse(stored) }` (defaults first, then stored values on top -- this preserves user edits while adding new default fields in future deploys).
- If not found or parse fails, use `DEFAULT_COMPANY_PROFILE`.
- Wrap all localStorage access in try/catch -- fall back to defaults silently on error (Safari private mode, quota exceeded).

**updateField function (useCallback):**
- Generic typed: `<K extends keyof CompanyProfile>(key: K, value: CompanyProfile[K]) => void`.
- Sets profile state with updated field.
- Persists entire profile to localStorage as JSON string.
- Wrap localStorage.setItem in try/catch -- fail silently.

**Key behavior:**
- Auto-save happens per-field when updateField is called (NOT on every keystroke -- the Settings page will use onBlur for text inputs to avoid excessive writes).
- Returns the full profile object and the updateField function.

```tsx
function ProfileField({ label, value, onChange }: { label: string; value: string; onChange: (v: string) => void }) {
  const [local, setLocal] = useState(value);
  return (
    <div>
      <label className="block text-xs font-medium text-slate-500 uppercase tracking-wider mb-1">{label}</label>
      <input
        type="text"
        value={local}
        onChange={(e) => setLocal(e.target.value)}
        onBlur={() => onChange(local)}
        className="w-full px-3 py-2 border border-slate-200 rounded-lg text-sm text-slate-900 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-transparent"
      />
    </div>
  );
}
```

Key: Use local state + onBlur to avoid excessive localStorage writes (per Pitfall 3 in RESEARCH.md). The onChange prop calls updateField which persists to localStorage. For date fields, use type="date" on the input and the same onBlur pattern.
