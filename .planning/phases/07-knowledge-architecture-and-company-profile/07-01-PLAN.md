**src/knowledge/registry.ts:**
1. Import KnowledgeModule from './types'.
2. Export `PASS_KNOWLEDGE_MAP: Record<string, string[]>` -- an object with all 16 pass names as keys, each mapping to an empty array `[]`. Use the exact pass name strings: 'risk-overview', 'dates-deadlines', 'scope-of-work', 'legal-indemnification', 'legal-payment-contingency', 'legal-liquidated-damages', 'legal-retainage', 'legal-insurance', 'legal-termination', 'legal-flow-down', 'legal-no-damage-delay', 'legal-lien-rights', 'legal-dispute-resolution', 'legal-change-order', 'verbiage-analysis', 'labor-compliance'.
3. Create a module-scoped `const moduleStore = new Map<string, KnowledgeModule>()`.
4. Export `registerModule(mod: KnowledgeModule): void` -- sets module in store by id.
5. Export `getModulesForPass(passName: string): KnowledgeModule[]` -- looks up IDs from PASS_KNOWLEDGE_MAP (default to empty array if pass not found), maps each ID to its module from moduleStore. Throws Error if a mapped module ID is not found in the store (developer error -- module not registered).

**src/knowledge/index.ts:**
1. Re-export all from './types' (KnowledgeModule, CompanyProfile, DEFAULT_COMPANY_PROFILE).
2. Re-export registerModule, getModulesForPass, PASS_KNOWLEDGE_MAP from './registry'.
3. Re-export estimateTokens, validateTokenBudget from './tokenBudget'.
4. Create and export `composeSystemPrompt(basePrompt: string, passName: string, companyProfile?: CompanyProfile): string`:
   - Call getModulesForPass(passName) to get relevant modules.
   - Call validateTokenBudget(modules) -- hard reject throws before any API call.
   - Start with basePrompt.
   - If modules.length > 0, append `\n\n## Domain Knowledge\n` header, then for each module append `\n### {title} (effective: {effectiveDate})\n{content}\n`.
   - If companyProfile is provided, append `\n\n## Company Profile\n` header, then call a local `formatCompanyProfile(profile)` function that formats as: Insurance line, Bonding line, License line, DIR line, SBE line, Capabilities line (see RESEARCH.md for exact format).
   - Return composed string.

Also create empty directories: `src/knowledge/regulatory/`, `src/knowledge/trade/`, `src/knowledge/standards/`. Add a `.gitkeep` file in each so git tracks them.

Do NOT modify api/analyze.ts -- Phase 8 wires composeSystemPrompt into the pipeline. Phase 7 only builds and exports.
