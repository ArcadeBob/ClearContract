## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge container `gap-1` |
| sm | 8px | Compact element spacing, pill `px-2` horizontal padding, badge container `mt-2` |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none. Pill vertical padding (`py-0.5` / 2px) is inherited from the existing shared `pillBase` utility and is not introduced by this phase.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label / Pill / Heading | 12px (`text-xs`) for pills, 18px (`text-lg`) for headings | 500 (`font-medium`) | 1.5 (pills), 1.2 (headings) |

This phase uses 2 weights: 400 (body text) and 500 (pill labels via `pillBase` `font-medium`). The broader codebase also uses 600 (`font-semibold`) for headings, but Phase 61 components do not introduce or require weight 600 -- all new text elements use `pillBase` which specifies `font-medium` (500).

Source: `ScopeMetaBadge/shared.ts` defines `pillBase` with `text-xs font-medium`.

### Phase 61 Badge Color Assignments

These colors are the specific pill background/text pairs for the two new badge components:

**WarrantyBadge pills:**

| Pill | Background | Text | Rationale |
|------|------------|------|-----------|
| Warranty Aspect (primary) | `bg-emerald-100` | `text-emerald-700` | Emerald signals compliance/coverage topic; consistent with BidReconciliationBadge primary pill |
| Warranty Duration (secondary, conditional) | `bg-blue-100` | `text-blue-700` | Blue for factual metadata; consistent with SpecReconciliationBadge secondary pill |
| Affected Party (tertiary, conditional) | `bg-slate-100` | `text-slate-700` | Slate for neutral attribution metadata; consistent with BidReconciliationBadge secondary pill |

**SafetyOshaBadge pills:**

| Pill | Background | Text | Rationale |
|------|------------|------|-----------|
| Safety Aspect (primary) | `bg-red-100` | `text-red-700` | Red signals safety/hazard domain; visually distinct from all other badge types |
| Regulatory Reference (secondary, conditional) | `bg-amber-100` | `text-amber-700` | Amber for regulatory/warning context; consistent with ExclusionStressTestBadge primary pill |
| Responsible Party (tertiary, conditional) | `bg-slate-100` | `text-slate-700` | Slate for neutral attribution metadata |

Source: Color pairings derived from existing badge components to maintain visual system consistency. Red-100/700 is new to badges but consistent with severity color system (`red=Critical`). Each pass gets a visually distinct primary pill color to aid scanning.

## Interaction Contracts

No new interactions in this phase. Both badge components are read-only display elements rendered within existing FindingCard components. The existing interaction patterns apply:

- Findings appear in the Findings tab under `Contract Compliance` category
- Category filter already supports `Contract Compliance` (existing category)
- Finding expand/collapse, action buttons, severity badges -- all existing
- Framer Motion stagger animation on finding list entry -- existing
- Badge pills are non-interactive (no hover state, no click handler)
