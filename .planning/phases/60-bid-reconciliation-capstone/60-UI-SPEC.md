### Phase 60 Document Attribution Colors

These colors are the critical UI contract for this phase. They distinguish quotes from different source documents.

| Document Source | Border Color | Background | Text Color | Usage |
|-----------------|-------------|------------|------------|-------|
| Contract | `border-slate-300` | `bg-slate-50` | `text-slate-700` | ClauseQuote default, existing pattern |

## Interaction Contract

### Bid-Reconciliation Finding States

| State | Condition | Behavior |
|-------|-----------|----------|
| Both quotes present | `contractQuote !== null && bidQuote !== null` | Render two ClauseQuote blocks stacked vertically |
| Contract only | `contractQuote !== null && bidQuote === null` | Render single Contract Language quote; bid is silent |
| Bid only | `contractQuote === null && bidQuote !== null` | Render single Bid Language quote; contract is silent |
| Neither quote | `contractQuote === null && bidQuote === null` | No quote blocks; finding still shows `clauseText` fallback |
| No bid uploaded | Pass skipped at orchestration layer | Zero bid-reconciliation findings in results; no UI trace |

### No New Pages or Navigation

Phase 60 adds no new views, pages, tabs, or navigation items. All new UI renders within existing FindingCard instances on the existing ContractReview page.
