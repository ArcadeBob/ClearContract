## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`) |
| lg | 24px | Section padding (`p-6`) |
| xl | 32px | Layout gaps (`gap-8`) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level spacing |

Exceptions: Table cells use `px-3 py-2` (12px horizontal, 8px vertical) to match compact data-table density. This is consistent with the existing DateTimeline row spacing pattern.

## Component Inventory

### New Components

#### 1. SubmittalRegister

- **Location:** `src/components/SubmittalRegister.tsx`
- **Props:** `submittals: SubmittalEntry[]`, `conflictFindings: Finding[]`
- **Layout:** White card (`bg-white rounded-lg border border-slate-200`) containing a responsive table.
- **Table columns:**

| Column | Width | Content |
|--------|-------|---------|
| Type | 120px fixed | Submittal type badge (`shop-drawing`, `sample`, `mockup`, `product-data`) |
| Description | flex-1 (min 200px) | Submittal description text |
| Duration | 80px fixed | `reviewDuration` days, or "---" in `text-slate-400 italic` if 0 and not in statedFields |
| Cycles | 64px fixed | `reviewCycles` count |
| Party | 140px fixed | `responsibleParty` name |
| Spec Section | 100px fixed | CSI reference (e.g., "08 44 13") |
| Status | 48px fixed | Warning icon if conflicted, empty otherwise |

- **Row behavior:**
  - Default row: `border-b border-slate-100` with `hover:bg-slate-50`
  - Conflicted row: `bg-amber-50 border-l-2 border-l-amber-400` with `AlertTriangle` icon in Status column
  - Clicking a conflicted row expands an inline detail panel showing: overrun days, milestone name, total duration breakdown, and any assumed values labeled explicitly
- **Empty state:** Never rendered (tab is hidden when submittals array is empty per CONTEXT.md)
- **Responsive:** Below 768px, table switches to stacked card layout with label-value pairs

#### 2. SubmittalTypeBadge

- **Props:** `type: SubmittalEntry['type']`

## Interaction Contracts

### Tab Selection

| Interaction | Behavior |
|-------------|----------|
| Click Submittals tab | `setViewMode('submittals')`, renders SubmittalRegister. Filter dropdowns (category, severity, priority) hide since they do not apply to register data. |
| Click any other tab | Returns to findings/negotiation view. Filter dropdowns reappear. |
| Tab hidden condition | `contract.submittals` is undefined, null, or empty array. Tab button does not render. |

### Conflict Row Expansion

| Interaction | Behavior |
|-------------|----------|
| Click conflicted row | Toggles inline expansion panel below the row (Framer Motion `AnimatePresence` + height animation). |
| Expansion content | Shows: "{description} requires {totalDays} total days ({reviewDuration}d review x {reviewCycles} cycles + {buffer}d buffer + {leadTime}d lead). This pushes {overrunDays} days past the '{milestoneName}' milestone." Assumed values labeled with "(assumed, not in contract)" in `text-amber-600 text-xs`. |
| Click expanded row again | Collapses the panel. |
| Multiple expansions | Multiple rows can be expanded simultaneously. |

### Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| >= 768px | Full table with horizontal scroll if needed (`overflow-x-auto`) |
| < 768px | Stacked card layout: each submittal renders as a card with label-value pairs stacked vertically. Type badge at top, conflict warning at bottom. |

## Animation Contract

| Element | Animation | Timing |
|---------|-----------|--------|
| Submittals tab appearance | None (conditional render, no transition) | Instant |
| SubmittalRegister rows | Staggered entry: `opacity 0->1, y 10->0` | `duration: 0.2, delay: index * 0.03` (faster than findings due to higher density) |
| Conflict expansion panel | Height expand from 0 + opacity fade in | `duration: 0.2, ease: easeOut` via Framer Motion `AnimatePresence` |
| Conflict expansion collapse | Height shrink to 0 + opacity fade out | `duration: 0.15, ease: easeIn` |

Source: Stagger pattern matches FindingCard (`index * 0.05`) but tightened to `0.03` because register rows are visually smaller. Expansion animation matches existing accordion patterns in the codebase.
