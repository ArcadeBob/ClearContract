## Interaction Contracts

### Subcategory Grouping (UX-01)

- Subcategory headers render ONLY within the "Scope of Work" CategorySection
- Subcategory headers are collapsible (click to expand/collapse), default expanded
- Subcategory headers show a finding count badge: "(N findings)"
- Subcategory grouping activates only when findings span 2+ distinct `sourcePass` values -- single-pass findings render flat (no sub-header noise)
- Subcategory header style: `text-xs font-medium text-slate-500 uppercase tracking-wide` with a left border accent (`border-l-2 border-violet-300 pl-3`)
- Finding cards within subcategories use `ml-2` indent (matches existing CategorySection `ml-2 mb-6` pattern)

### Scope Intelligence View-Mode (UX-02)

- View-mode button appears in FilterToolbar ONLY when contract has scope-intel data (`submittals.length > 0` OR findings with `sourcePass` in the scope-intel pass set)
- Three sub-components stack vertically with `space-y-6` gap
- Each sub-component is wrapped in a white card panel: `bg-white rounded-xl border border-slate-200 shadow-sm p-6`
- Each sub-component has its own section heading: `text-sm font-semibold text-slate-900` with icon in `p-2 rounded-lg` colored background
- Sub-component order: Submittal Timeline (top), Spec Gap Matrix (middle), Bid/Contract Diff (bottom)
- Filter dropdowns are hidden when `viewMode === 'scope-intel'` (same pattern as `viewMode === 'submittals'`)

### Submittal Timeline

- Vertical layout: each submittal as a row with a horizontal duration bar
- Duration bar width proportional to total days (review period + resubmittal buffer)
- Duration bar color: `bg-blue-200` default, `bg-amber-200` when schedule-conflict exists for that submittal
- Contract milestone markers shown as vertical dashed lines (`border-dashed border-slate-300`) with date labels
- Conflict indicator: `AlertTriangle` icon in amber next to conflicting submittals
- "Not stated" durations shown with `text-slate-400 italic` text ("---") -- reuses SubmittalRegister pattern

### Spec Gap Matrix

- Table layout with rows grouped by spec standard source (AAMA, ASTM, Div 08)
- Columns: Spec Reference, Expected Deliverable, Status (Present / Gap / Inferred), Severity badge
- "Gap" status cells: `bg-amber-50 text-amber-700`
- "Present" status cells: `bg-emerald-50 text-emerald-700`
- "Inferred" status cells: `bg-blue-50 text-blue-700` with inference-basis pill
- Row click expands to show finding detail (clause quote + explanation)

### Bid/Contract Diff

- Three grouped sections: Exclusion Parity, Quantity Deltas, Unbid Scope
- Each section has a count badge: "(N items)"
- Each finding row shows dual quotes: contract quote (left, `border-l-2 border-slate-300`) and bid quote (right, `border-l-2 border-emerald-300`)
- Missing quote (one document silent): `text-slate-400 italic` "Not addressed in [contract/bid]"
- Direction-of-risk text truncated at 60 characters with ellipsis (Phase 60-02 decision)

### Scope Trends Card (PORT-01)

- Card style matches PatternsCard: `bg-white rounded-xl border border-slate-200 shadow-sm p-6`
- Icon: `TrendingUp` (Lucide) in `bg-violet-50 text-violet-600 p-2 rounded-lg`
- Three trend sections stacked vertically with `space-y-4`
- Each trend section: label in `text-xs font-medium text-slate-500 uppercase tracking-wide`, then ranked list (top 5 items)
- Each ranked item: `text-sm text-slate-700` name + `text-xs font-medium text-slate-500` count suffix ("N/M contracts")
- Card returns `null` when fewer than 10 reviewed contracts -- no empty state, just absent
- Framer Motion entry animation: `initial={{ opacity: 0, y: 12 }} animate={{ opacity: 1, y: 0 }} transition={{ delay: 0.2 }}` (matches PatternsCard)

### Legacy Contract Handling

- Pre-v3.0 contracts: Scope Intel button hidden in FilterToolbar (no submittals, no scope-intel sourcePass)
- Pre-v3.0 contracts: CategorySection "Scope of Work" renders flat (no subcategory headers) since all findings have null/undefined `sourcePass` mapping to a single group
- ScopeTrendsCard: only counts contracts with `status === 'Reviewed' || status === 'Partial'`; pre-v3.0 contracts contribute zero scope-intel findings to aggregation (correct behavior)
