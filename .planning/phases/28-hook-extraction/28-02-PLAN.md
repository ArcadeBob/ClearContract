Options interface:
```typescript
interface UseFieldValidationOptions {
  initialValue: string;
  validate: (value: string) => { valid: boolean; error?: string; warning?: string; formatted?: string };
  onSave: (value: string) => void;
}
```

Return interface:
```typescript
interface UseFieldValidationReturn {
  inputProps: {
    value: string;
    onChange: (e: React.ChangeEvent<HTMLInputElement>) => void;
    onFocus: () => void;
    onBlur: () => void;
  };
  error: string | null;
  warning: string | null;
  showSaved: boolean;
}
```

Implementation (extract from Settings.tsx ProfileField lines 23-77):
- `localValue` state initialized to `initialValue`
- `error` and `warning` state (string | null)
- `showSaved` state (boolean)
- `focusedRef` (useRef<boolean>) -- tracks focus to prevent external sync during editing
- `timerRef` (useRef<ReturnType<typeof setTimeout>>) -- for Saved flash cleanup

Effects:
- Sync effect: when `initialValue` changes and `focusedRef.current` is false, update `localValue` to `initialValue`
- Cleanup effect: clear timerRef on unmount

Handlers (all wrapped in useCallback with correct deps):
- `onChange`: set localValue from event, clear error
- `onFocus`: set focusedRef true, clear error
- `onBlur`: set focusedRef false, run validate(localValue). If invalid: set error, revert localValue to initialValue. If valid: clear error, set warning, apply formatted value if present, call onSave if value changed, trigger showSaved=true with 2-second timeout

Return `{ inputProps: { value: localValue, onChange, onFocus, onBlur }, error, warning, showSaved }`

Export as named export: `export function useFieldValidation(options: UseFieldValidationOptions): UseFieldValidationReturn`
