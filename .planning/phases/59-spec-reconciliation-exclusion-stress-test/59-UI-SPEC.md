### Exclusion Stress-Test: Dual-Quote Display

The exclusion stress-test pass produces two quoted texts per finding: the exclusion clause and the tension requirement. These render as a pair of `ClauseQuote`-style blocks within the existing `FindingCard` expanded view:

| Quote Block | Label Copy | Border Color |
|-------------|------------|--------------|
| Exclusion quote | "Declared Exclusion -- {clauseReference}" | `border-slate-300` (existing `ClauseQuote` style) |
| Tension quote | "Inferred Requirement -- {specSection}" | `border-amber-300` (distinguishes inference from contract language) |

The tension quote block uses `border-amber-300` instead of the default `border-slate-300` to visually signal that this is an inference-based citation, not a direct contract quote. Background remains `bg-slate-50` for both.

Source: `ClauseQuote.tsx` existing pattern, REQUIREMENTS.md SCOPE-04 ("both the exclusion quote and the tension quote displayed").

### FindingCard Extension

The `FindingCard` component already renders `scopeMeta` via `ScopeMetaBadge`. No structural changes needed. The dual-quote display for exclusion-stress-test findings requires a conditional block in `FindingCard` that checks `finding.scopeMeta?.passType === 'exclusion-stress-test'` and renders:

1. Standard `ClauseQuote` for `clauseText` (the exclusion) -- uses existing `border-slate-300`
2. A second quote block for `tensionQuote` with label "Inferred Requirement" -- uses `border-amber-300` background variant

This second quote block reuses the `ClauseQuote` layout but with the amber border. Implementation options:
- **Option A (recommended):** Add an optional `borderColor` prop to `ClauseQuote` defaulting to `border-slate-300`
- **Option B:** Create a standalone `InferenceQuote` component duplicating the layout with amber border

## Phase Scope -- UI Surface Summary

This phase is primarily backend (two new LLM passes, schemas, merge handlers). The frontend surface is minimal and confined to:

1. **Two new ScopeMetaBadge variants** -- pill rows within existing `FindingCard` for the two new `passType` values
2. **Dual-quote display** -- a second `ClauseQuote`-style block with amber border for exclusion-stress-test tension quotes
3. **PASS_LABELS entries** -- two new labels in `CostSummaryBar` for the cost summary display
4. **ScopeMeta union extension** -- two new variants in `src/types/contract.ts` (no visual impact, enables badge dispatch)

No new pages. No new navigation. No new views. No layout changes. All new findings render through the existing `FindingCard` + `CategorySection` + `ScopeMetaBadge` pipeline under the "Scope of Work" category.
