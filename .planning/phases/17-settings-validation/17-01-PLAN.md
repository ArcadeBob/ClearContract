  <name>Task 1: Create validation utility module</name>
  <files>src/utils/settingsValidation.ts</files>
  <action>
Create `src/utils/settingsValidation.ts` with pure validation and formatting functions. No React dependencies.

Define types:
- `FieldType = 'dollar' | 'date' | 'employeeCount' | 'text'`
- `ValidationResult = { valid: boolean; error?: string; warning?: string; formatted?: string }`

Implement these exported functions:

1. `validateDollar(raw: string): ValidationResult` — Strip `$`, commas, whitespace. Empty/whitespace-only returns `{ valid: true }`. Check remaining is digits only (optional decimal: `/^\d+(\.\d+)?$/`). If not, return `{ valid: false, error: 'Enter a dollar amount (e.g., $1,000,000)' }`. If valid, format using `Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD', minimumFractionDigits: 0, maximumFractionDigits: 0 })` and return `{ valid: true, formatted }`. $0 is valid.

2. `validateDate(raw: string): ValidationResult` — Empty returns `{ valid: true }`. Parse with `new Date(raw + 'T00:00:00')`. If `isNaN`, return `{ valid: false, error: 'Enter a valid date' }`. If date is before today (compare at midnight), return `{ valid: true, warning: 'This date has passed' }`. Otherwise `{ valid: true }`.

3. `validateEmployeeCount(raw: string): ValidationResult` — Empty returns `{ valid: true }`. Accept `/^\d+(\s*-\s*\d+)?$/`. If valid, normalize spaces around dash (`trimmed.replace(/\s*-\s*/, '-')`) and return `{ valid: true, formatted }`. If invalid, return `{ valid: false, error: 'Enter a number or range (e.g., 15-25)' }`.

4. `validateField(value: string, fieldType: FieldType): ValidationResult` — Dispatcher. Empty string always returns `{ valid: true }`. Switch on fieldType to call the appropriate validator. 'text' always returns `{ valid: true }`.

Export: `validateField`, `FieldType`, `ValidationResult`.

  <name>Task 2: Enhance ProfileField with validation, formatting, and save feedback</name>
  <files>src/pages/Settings.tsx, src/hooks/useCompanyProfile.ts</files>
  <action>
**useCompanyProfile.ts changes:**

Add a `saveField` method alongside existing `updateField`. `saveField` checks if the value actually changed (`prev[key] === value` returns prev unchanged). If changed, spreads the new value and persists to localStorage (same pattern as updateField). Return `{ profile, updateField, saveField }`.

**Settings.tsx changes:**

1. **Add fieldType to CardSection fields interface:**
   Change the `fields` type in `CardSection` to include `fieldType?: FieldType` (import `FieldType` from `../utils/settingsValidation`). Default is `'text'`.

2. **Add fieldType metadata to each card definition:**
   - Insurance Coverage: `glPerOccurrence`, `glAggregate`, `umbrellaLimit`, `autoLimit`, `wcEmployerLiability` get `fieldType: 'dollar'`. `wcStatutoryState` stays text (no fieldType needed).
   - Bonding Capacity: `bondingSingleProject`, `bondingAggregate` get `fieldType: 'dollar'`.
   - Licenses & Certifications: `contractorLicenseExpiry` and `dirExpiry` get `fieldType: 'date'`. All others stay text.
   - Company Capabilities: `employeeCount` gets `fieldType: 'employeeCount'`. `typicalProjectSizeMin`, `typicalProjectSizeMax` get `fieldType: 'dollar'`. `serviceArea` stays text.

3. **Rewrite ProfileField** to support validation, local state, and save feedback:

   New props: `{ label, value, onSave, type?, fieldType? }` — rename `onChange` to `onSave` (called only on valid blur).

   Internal state:
   - `localValue` (useState initialized from `value` prop)
   - `error` (string | null)
   - `warning` (string | null)
   - `showSaved` (boolean)

   Add `useEffect` to sync `localValue` when `value` prop changes externally (but skip if field is focused — track focus with a `useRef<boolean>`).

   `onChange` handler: set `localValue` freely (user types without restriction). Clear `error` on any change (per decision: "errors clear immediately when user corrects the value").

   `onFocus` handler: set focused ref to true. Clear `error` state.

   `onBlur` handler (the core flow):
   - Set focused ref to false.
   - Call `validateField(localValue, fieldType || 'text')` from the utility.
   - If `!result.valid`: set `error` to `result.error`, revert `localValue` to `value` prop (last saved). Return early — do NOT call onSave.
   - If valid: clear `error`, set `warning` to `result.warning || null`. Determine `finalValue = result.formatted || localValue`. Update `localValue` to `finalValue`.
   - If `finalValue !== value` (actually changed): call `onSave(finalValue)`, set `showSaved = true`, start a 2-second `setTimeout` to set `showSaved = false`. Store timer ID in a ref, clear on unmount via `useEffect` cleanup.
   - If `finalValue === value`: no-op (no save indicator).

   Render:
   - Input gets conditional border classes: `border-red-300 focus:ring-red-500` when `error` is set, normal `border-slate-200 focus:ring-blue-500` otherwise.
   - Below the input, render error message: `<p className="mt-1 text-xs text-red-500">{error}</p>` when `error` is set.
   - Below the input, render warning message: `<p className="mt-1 text-xs text-amber-500">{warning}</p>` when `warning` is set (and no error).
   - Next to the label (or after the input), render the saved indicator using Framer Motion AnimatePresence:
     ```tsx
     <AnimatePresence>
       {showSaved && (
         <motion.span
           initial={{ opacity: 0 }}
           animate={{ opacity: 1 }}
           exit={{ opacity: 0 }}
           className="inline-flex items-center gap-1 text-xs text-emerald-600"
         >
           <Check className="w-3 h-3" />
           Saved
         </motion.span>
       )}
     </AnimatePresence>
     ```
   - Import `Check` from `lucide-react`, `AnimatePresence` from `framer-motion`, `validateField` and `FieldType` from utils.

4. **Update Settings component** to use `saveField` instead of `updateField`:
   - Destructure `{ profile, saveField }` from `useCompanyProfile()`.
   - Pass `onSave={(v) => saveField(field.key, v)}` and `fieldType={field.fieldType}` to each ProfileField.
