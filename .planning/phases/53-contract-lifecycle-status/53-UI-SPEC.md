## Visual Hierarchy

**Primary focal point:** The LifecycleSelect dropdown on the ReviewHeader metadata row. This is the only interactive lifecycle element on the contract review page and the primary action surface for this phase. It appears inline in the header metadata row after the contract type, visually anchored by its bordered select appearance against the flat text of the surrounding metadata.

**Secondary focal points:** LifecycleBadge on ContractCard (top-right badge stack) and the Lifecycle filter button on AllContracts toolbar.

## Component Inventory

### New Components

#### LifecycleBadge

- **Pattern:** Mirrors `SeverityBadge` exactly
- **Classes:** `inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-normal border` + color from palette
- **Props:** `status: LifecycleStatus`, `className?: string`
- **Renders:** Status text as label

#### LifecycleSelect

- **Element:** Native `<select>` (not a custom dropdown -- per research recommendation)
- **Classes:** `text-sm border border-slate-200 rounded-lg px-3 py-1.5 bg-white text-slate-700 focus:outline-none focus:ring-2 focus:ring-blue-500`
- **Props:** `current: LifecycleStatus`, `onChange: (status: LifecycleStatus) => void`
- **Behavior:** First option is current status (selected, acts as label). Remaining options are only valid transitions from `LIFECYCLE_TRANSITIONS[current]`. If current status is Expired (terminal), select is disabled.
- **Disabled state:** `opacity-50 cursor-not-allowed` when no valid transitions exist

### Modified Components

#### ContractCard

- **Change:** Add `LifecycleBadge` next to existing analysis status badge
- **Placement:** In the top-right column (`div.flex.flex-col.items-end`), below the date and above the existing analysis status span (line 68-77 of current component)
- **Spacing:** `mb-1` between lifecycle badge and analysis status badge
- **Visual hierarchy:** Lifecycle badge appears first (business status), analysis badge appears second (technical status)

#### ReviewHeader

- **Change:** Add `LifecycleSelect` in the header metadata area
- **Placement:** In the metadata row (line 99-103), after the client/type text, add a separator bullet and the LifecycleSelect dropdown
- **Layout:** `<span>{contract.client}</span> <span>&bull;</span> <span>{contract.type}</span> <span>&bull;</span> <LifecycleSelect current={contract.lifecycleStatus} onChange={...} />`
- **The select replaces a static badge on this page** since users need to change status here

#### AllContracts

- **Change:** Add lifecycle filter using existing `MultiSelectDropdown`
- **Placement:** In the filter toolbar, alongside existing filter controls
- **Props:** `label="Lifecycle"`, `options={LIFECYCLE_STATUSES}`, `selected={lifecycleFilter}`, `onChange={setLifecycleFilter}`
- **Default state:** All statuses selected (no filter active)
- **renderOption:** Each option renders with a small colored dot matching the badge color for that status, followed by the status name

## Interaction Contracts

### Status Transition (LifecycleSelect)

1. User clicks the select dropdown on ReviewHeader
2. Dropdown shows current status (selected) + valid transitions only
3. User selects a new status
4. Optimistic update: local state updates immediately, LifecycleBadge on all visible cards reflects new status
5. Supabase write fires in background
6. On success: no additional UI feedback needed (state already correct)
7. On failure: rollback to previous status, show error toast "Failed to update status. Changes reverted."

### Lifecycle Filter (AllContracts)

1. User clicks "Lifecycle" dropdown button in filter toolbar
2. MultiSelectDropdown opens showing all 6 statuses with checkboxes
3. Each status option shows a colored dot + status name
4. User checks/unchecks statuses to filter
5. Contract list filters in real-time (client-side, no API call)
6. Filter count badge shows "N/6" when not all selected
7. Escape key or click outside closes dropdown

### Terminal State (Expired)

1. When a contract's lifecycle status is "Expired", the LifecycleSelect is disabled
2. Visual: select shows reduced opacity (opacity-50) and not-allowed cursor
3. No transitions are available from Expired (it is a terminal state)
