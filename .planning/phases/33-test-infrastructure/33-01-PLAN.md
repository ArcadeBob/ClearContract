  <action>
1. Create `src/test/mocks/framer-motion.ts`:
   - Use `vi.mock('framer-motion', () => { ... })` with a Proxy-based mock
   - The Proxy handles all `motion.*` element access (motion.div, motion.span, etc.) dynamically
   - Each proxied component is a `React.forwardRef` that:
     - Filters out framer-motion-specific props: animate, initial, exit, transition, variants, whileHover, whileTap, whileFocus, whileInView, layout, drag, dragConstraints, onAnimationComplete
     - Passes remaining props + ref to `React.createElement(prop, { ...filteredProps, ref })`
     - Has displayName `motion.${prop}`
   - Uses a `Map<string, React.FC>` cache to avoid re-creating components on each access
   - Exports: `motion` (the Proxy), `AnimatePresence` (renders children directly: `({ children }) => children`), `useAnimation` (returns `{ start: vi.fn(), stop: vi.fn(), set: vi.fn() }`), `useMotionValue` (returns `{ get: () => initial, set: vi.fn(), onChange: vi.fn() }`), `useTransform` (returns `{ get: () => output?.[0] ?? 0, set: vi.fn(), onChange: vi.fn() }`)

2. Create `src/test/setup.ts`:
   ```typescript
   import '@testing-library/jest-dom/vitest';
   import './mocks/framer-motion';
   ```
   That is the complete file -- two imports. The jest-dom/vitest import auto-extends `expect` with all DOM matchers. The framer-motion import executes the vi.mock call globally.

3. Create `src/test/render.tsx`:
   - Import `render` and `RenderOptions` from `@testing-library/react`
   - Import `ReactElement` from `react`
   - Import `ToastProvider` from `../contexts/ToastProvider`
   - Create `AllProviders` component that wraps children in `<ToastProvider>{children}</ToastProvider>`
   - Create `customRender` function: `(ui: ReactElement, options?: Omit<RenderOptions, 'wrapper'>) => render(ui, { wrapper: AllProviders, ...options })`
   - Export `customRender as render`
   - Re-export `screen`, `within`, `waitFor` from `@testing-library/react`
   - Re-export `default as userEvent` from `@testing-library/user-event`
  </action>
