## Component Inventory

### New Components

#### 1. Bid Drop Zone (extends UploadZone pattern)

**Location:** Renders below the primary contract UploadZone on `ContractUpload` page.
**Visual treatment:** Same dashed-border drop zone pattern as contract zone, but visually subordinate:
- Border: `border-slate-200` (lighter than contract zone's `border-slate-300`)
- Background: `bg-slate-50/50` (subtly tinted vs. contract zone's white)
- Icon: `FileSpreadsheet` from lucide-react (distinct from `UploadCloud` on contract zone)
- Padding: `p-8` (smaller than contract zone's `p-12`)
- Corner badge: "Optional" in `text-xs text-slate-400 bg-slate-100 rounded-full px-2 py-0.5` positioned top-right of the zone

**States:**
| State | Visual |
|-------|--------|
| Empty (default) | Dashed border, "Add Bid / Estimate PDF" heading, "Optional -- attach your bid to enable cross-document analysis" body text |
| Drag active | `border-blue-500 bg-blue-50` (matches contract zone active state) |
| File selected | Solid `border-slate-300`, file name + size displayed, "Remove" text button in `text-red-500 hover:text-red-600 text-xs` |
| Error (rejection) | `border-red-400`, animated error message with AlertCircle icon (matches contract zone error pattern) |
| Hidden (analyzing) | Entire bid zone hidden during analysis -- only the analyzing spinner shows |

**Constraints:**
- Max file size: 5MB (enforced client-side; smaller than contract's 10MB)
- Accepted types: PDF only (`application/pdf`)
- File size hint: "PDF up to 5MB" (vs. contract zone's "PDF up to 10MB")

#### 2. Re-Analyze Document Selection Modal

**Location:** Portal to `document.body` (follows existing `ConfirmDialog` pattern).
**Trigger:** Existing re-analyze button on `ReviewHeader`.
**Layout:** Centered modal, `max-w-md w-full`, white background, rounded-xl, shadow-xl, p-6.

**Structure (top to bottom):**
1. **Header:** RefreshCw icon in blue-100 circle (matches existing re-analyze icon pattern), "Re-Analyze Contract" heading at text-lg font-semibold
2. **Contract section:** Label "Contract PDF" in text-sm text-slate-700, radio group:
   - "Keep current contract" (default selected)
   - "Upload new contract PDF" -- reveals inline file picker when selected
3. **Bid section:** Label "Bid / Estimate PDF" in text-sm text-slate-700, radio group:
   - "No bid attached" (default when contract has no bid) OR "Keep current bid" (when bid exists)
   - "Upload new bid PDF" -- reveals inline file picker when selected
   - "Remove bid" (only shown when bid exists) in text-red-500
4. **Actions:** "Keep Current Documents" secondary button + "Start Analysis" primary button (blue-600 bg, white text)

**Radio styling:** Custom radio using `w-4 h-4 text-blue-600` with labels in `text-sm text-slate-600`.
**Inline file picker:** When radio selects "Upload new...", a compact file input appears below with file name display and remove action.
**Backdrop:** `bg-black/50`, click to cancel. Escape key closes. Focus trap on open.

#### 3. Documents Analyzed Badge

**Location:** `ReviewHeader` component, next to contract metadata.
**When visible:** Only when `bidFileName` is present on the contract.
**Visual:** Inline flex badge: `bg-emerald-50 text-emerald-700 border border-emerald-200 text-xs px-2 py-0.5 rounded-full` with text "Contract + Bid".
**When no bid:** No badge renders (not an empty state -- simply absent).

### Modified Components

#### ContractUpload Page

**Current:** Single UploadZone centered in a max-w-2xl card.
**Change:** Add bid drop zone below the contract zone, separated by 24px gap (`gap-6` or `space-y-6`). The card width remains `max-w-2xl`. Page heading changes from "Upload Contract" to "Upload Documents". Subheading changes from "Upload a PDF contract to start the AI analysis process." to "Upload your contract PDF to start analysis. Optionally attach a bid for cross-document review."

**File state management:** Both files lifted to ContractUpload state:
```
contractFile: File | null
bidFile: File | null
```
Contract zone calls `setContractFile`. Bid zone calls `setBidFile`. The "Analyze" flow passes both to `onUploadComplete`.

**Analyze button visibility:** When `contractFile` is set, show an explicit "Analyze Contract" button below both zones: `bg-blue-600 text-white font-semibold text-sm px-6 py-3 rounded-lg hover:bg-blue-700 transition-colors w-full`. When `bidFile` is also set, button label changes to "Analyze Contract + Bid".

#### ReviewHeader

**Change:** Add documents-analyzed badge (see component 3 above). Wire re-analyze button to open the new document selection modal instead of the current direct file picker.

#### UploadZone

**Change:** Add `role` prop (`'contract' | 'bid'`) to customize heading, body text, icon, max file size, and visual weight. Default role is `'contract'` for backward compatibility.

## Interaction Contracts

### Upload Flow

1. User lands on upload page. Contract drop zone is prominent. Bid drop zone sits below, visually subordinate with "Optional" badge.
2. User drops or selects a contract PDF. Contract zone shows file name + size. "Analyze Contract" button appears below both zones.
3. (Optional) User drops or selects a bid PDF. Bid zone shows file name. Button label changes to "Analyze Contract + Bid".
4. User can remove either file independently. Removing contract hides the analyze button. Removing bid reverts button label.
5. User clicks analyze. Both zones and button replaced by analyzing spinner. Page navigates to review on completion.

### Re-Analyze Flow

1. User clicks re-analyze on ReviewHeader. Document selection modal opens.
2. Modal pre-populates based on current contract state (bid present or not).
3. User selects document configuration. "Start Analysis" button is always enabled (at minimum, the contract is kept).
4. On confirm, modal closes, review page shows re-analyzing banner, analysis proceeds.

### Backward Compatibility

- Contracts analyzed before Phase 58 have no `bidFileName`. The documents badge does not render. Re-analyze modal shows "No bid attached" as default for bid section.
- No UI elements break or show error states for pre-v3.0 contracts.
