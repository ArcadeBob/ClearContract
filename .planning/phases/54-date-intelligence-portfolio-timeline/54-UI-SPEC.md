## Typography

| Role | Size | Weight | Line Height |

### Urgency Group Color Palette

These are the specific color assignments for the four urgency buckets. Source: 54-RESEARCH.md Pattern 1.

| Group | Text Color | Background | Icon Color | Rationale |
|-------|-----------|------------|------------|-----------|
| Overdue | text-red-600 | bg-red-50 | text-red-500 | Matches Critical severity convention (red = danger) |
| This Week | text-amber-600 | bg-amber-50 | text-amber-500 | Matches High severity convention (amber = warning) |
| This Month | text-blue-600 | bg-blue-50 | text-blue-500 | Matches Low severity convention (blue = informational) |
| Later | text-slate-600 | bg-slate-50 | text-slate-400 | Matches Info severity convention (slate = neutral) |

Accent reserved for: Primary CTA button, active sidebar nav indicator, "This Month" urgency icon accent. Never for urgency groups "Overdue" or "This Week" (those use semantic red/amber).

### Sidebar Deadline Badge Color

| Element | Classes | Rationale |
|---------|---------|-----------|
| Deadline badge (>0) | bg-red-600 text-white text-xs py-0.5 px-2 rounded-full | Differentiates from existing contract count badge (bg-slate-800 text-slate-300). Red signals urgency. |
| Deadline badge (0) | Not rendered | Badge prop set to undefined when count is 0. Existing truthy check hides it. |

## Component Inventory

### New Components

#### DeadlineTimeline

- **Location:** `src/components/DeadlineTimeline.tsx`
- **Purpose:** Portfolio-wide deadline timeline grouped by urgency (DATE-01)
- **Container:** `bg-white rounded-xl border border-slate-200 shadow-sm p-6` (matches existing Dashboard card pattern)
- **Section headers:** Each urgency group gets a header row with icon + label using the urgency group color palette above
- **Section header layout:** `flex items-center gap-2 mb-3 mt-4` with group icon (Calendar for overdue/this-week, Clock for this-month/later) and label in group text color
- **First section header:** No `mt-4` (no top margin on first group)
- **Timeline entries:** Clickable button, `w-full text-left p-3 rounded-lg hover:bg-slate-50 transition-colors border border-slate-100` (matches existing upcoming dates widget entry pattern exactly)
- **Entry row 1:** Date label (text-sm font-semibold text-slate-900 truncate) + urgency relative label on right (text-xs font-semibold in group text color)
- **Entry row 2:** Contract name (text-xs text-slate-500 truncate) + formatted date on right (text-xs text-slate-500)
- **Entry click:** Calls `onNavigate('review', contractId)` -- navigates to that contract's review page
- **Empty groups:** Not rendered. Only groups with at least one entry display their header + entries.
- **Animation:** Framer Motion staggered entry on each entry item: `initial={{ opacity: 0, y: 8 }} animate={{ opacity: 1, y: 0 }} transition={{ delay: index * 0.03 }}` (faster than standard 0.05 since timeline may have many entries)

#### Empty State (within DeadlineTimeline)

- **When:** Zero dates across all contracts (all groups empty)
- **Layout:** Centered vertically within the card, `text-center py-8`
- **Icon:** `Calendar` from Lucide, `w-12 h-12 text-slate-200 mx-auto mb-3`
- **Heading:** "No upcoming deadlines" in text-sm text-slate-400
- **Body:** "Upload and analyze contracts to track important dates" in text-xs text-slate-400 mt-1

### Modified Components

#### Sidebar

- **Change:** Accept new `deadlineBadge` number prop
- **Badge placement:** On the Dashboard nav item (id: 'dashboard')
- **Badge style:** `bg-red-600 text-white text-xs py-0.5 px-2 rounded-full` (distinct from existing slate contract count badge)
- **Visibility:** Only shown when `deadlineBadge > 0`. Pass `undefined` when 0.

#### Dashboard

- **Change:** Replace existing "Upcoming Deadlines" widget (lines 245-282) with new `DeadlineTimeline` component
- **Position:** Same right-column position, below "Quick Actions"
- **The old `getDateUrgency` function (lines 21-42) and `upcomingDates` memo (lines 66-79) are removed.** Urgency logic moves to `src/utils/dateUrgency.ts`.

#### App.tsx

- **Change:** Compute 7-day deadline count from contracts array using `countDeadlinesWithin7Days` utility
- **Threading:** Pass computed count as `deadlineBadge` prop to Sidebar

## Interaction Contract

### Timeline Entry Click

| Property | Value |
|----------|-------|
| Target | Each timeline entry row |
| Action | `onNavigate('review', entry.contractId)` |
| Feedback | `hover:bg-slate-50` background transition |
| Cursor | `cursor-pointer` (implicit on button element) |

### Sidebar Badge

| Property | Value |
|----------|-------|
| Visibility | Shown when deadline count within 7 days > 0 |
| Hidden | When count is 0 (prop set to undefined) |
| Content | Integer count of deadlines due in next 7 days |
| Updates | Reactively via useMemo when contracts array changes |

### Urgency Grouping Rules

| Group | Condition |
|-------|-----------|
| Overdue | diffDays < 0 AND diffDays >= -30 (last 30 days only, per RESEARCH.md recommendation) |
| This Week | diffDays >= 0 AND diffDays <= 7 |
| This Month | diffDays > 7 AND diffDays <= 30 |
| Later | diffDays > 30 |

Dates older than 30 days overdue are excluded from the timeline entirely. They are no longer actionable.

## Layout Contract

### DeadlineTimeline Position on Dashboard

The Dashboard uses a two-column layout for the lower section:
- **Left column (col-span-2):** Recent Contracts list
- **Right column (col-span-1):** Quick Actions + DeadlineTimeline (replaces old Upcoming Deadlines widget)

The DeadlineTimeline card sits directly below the Quick Actions card in the right column, separated by `space-y-6` (24px).

### DeadlineTimeline Internal Layout

```
+------------------------------------------+
| [Calendar icon] Portfolio Deadlines       |  <-- Card header (text-lg font-semibold)
+------------------------------------------+
| [AlertCircle] Overdue                     |  <-- Group header (bg-red-50 rounded-lg px-3 py-1.5)
|  [entry] Retainage release    3d ago      |
|          ABC Construction  Mar 19, 2026   |
|  [entry] Insurance cert due   7d ago      |
|          XYZ Glazing       Mar 15, 2026   |
+------------------------------------------+
| [Clock] This Week                         |  <-- Group header (bg-amber-50 rounded-lg px-3 py-1.5)
|  [entry] Submittal deadline   2d away     |
|          DEF Contract      Mar 24, 2026   |
+------------------------------------------+
| [Calendar] This Month                     |  <-- Group header (bg-blue-50 rounded-lg px-3 py-1.5)
|  [entry] Progress payment    18d away     |
|          GHI Project       Apr 9, 2026    |
+------------------------------------------+
```

Group headers use inline flex with rounded background pill: `flex items-center gap-2 px-3 py-1.5 rounded-lg {bgClass}` with the icon and label in the group's text color.
